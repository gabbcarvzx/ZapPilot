# ZapPilot Local v1.0.0 First Client Ready

## Resumo Executivo

Esta release congela a primeira versao comercial utilizavel do `ZapPilot Local` para ativacao do primeiro cliente pagante.

O foco desta entrega foi:

- tornar a ativacao operacionalmente guiada
- reduzir contradicoes de status no produto
- melhorar a leitura comercial do onboarding
- criar dados demo reproduziveis para calls de venda
- deixar o painel mais honesto sobre o que bloqueia ou libera operacao real
- formalizar o script de validacao ponta a ponta

Estado final da release:

- deploy validado
- testes, lint e build verdes
- fluxo admin de ativacao legivel
- simulador e onboarding prontos para demo comercial
- diagnostico por tenant coerente com o caminho operacional atual

## Funcionalidades Disponiveis

- landing page comercial, signup, login e painel protegido
- ativacao manual de plano via `/admin`
- diagnostico operacional com plano, WhatsApp, IA, banco e webhook
- onboarding guiado em 3 passos
- configuracao de negocio, WhatsApp e atendente
- mensagens simuladas em modo mock
- suporte a validacao live quando credenciais e numero do cliente estiverem prontos
- seed demo com tres tenants comerciais
- persistencia real com Prisma + Neon quando `DATABASE_URL` esta configurada
- fallback previsivel para ambientes sem integracoes reais

## Limitacoes Atuais

- billing continua manual e admin-driven
- uso comercial live do WhatsApp ainda depende do numero do cliente estar pronto na Meta
- paginas legais `/privacy` e `/terms` continuam temporarias
- fallback humano e quota da Gemini ainda dependem de operacao manual e monitoramento
- nao existe automacao avancada de observabilidade, analytics ou RBAC empresarial
